# Skill Review v3.0 — 收费产品级质检

## 环境与兜底
- 目标目录必须存在且可读；UTF-8 编码；PowerShell 环境
- 每个 skill 须有 SKILL.md（否则标记 INCOMPLETE）
- 禁止递归评测自身
- 单文件读取失败: 标记 INCOMPLETE，不中断全量扫描
- 输出报告路径: `<目标目录>/_reports/skill-review-YYYYMMDD-HHmm.md`

## 预扫描检查清单（Step 0，任一❌=中止并报告原因）
- [ ] 目标目录存在: `Test-Path <dir>`
- [ ] 至少有 1 个 SKILL.md: `(Get-ChildItem -Recurse -Filter SKILL.md).Count -gt 0`
- [ ] 上次报告目录可写: `Test-Path <dir>/_reports/` 或可创建
- [ ] 输出路径无同名文件冲突(若有→自动追加-v2后缀)

## 10 维评分体系（收费产品级校准）

**通用刻度**: 10=满分无缺陷 | 8-9=基本满足(1-2小缺口) | 5-7=部分满足(≥1明显缺失) | 2-4=严重不足 | 0-1=缺失

| # | 维度(W) | 0分 | 5分 | 10分(满分) |
|---|--------|-----|-----|-----------|
| D1 | 触发精准(10%) | 无description/无触发词 | <5触发词,缺不适用场景 | ≥5触发词+不适用场景+≥2正反例+边界 |
| D2 | 结构完整(10%) | 无frontmatter, body空 | 缺操作步骤/验证/凭据之一 | frontmatter+步骤序列+布尔验证+故障处理表 |
| D3 | 可执行(15%) | 全是模糊建议词 | 1-2条可执行,其余模糊 | 每条含具体动作(命令/参数/布尔条件),可复制执行 |
| D4 | 验证闭环(10%) | 无验收标准 | 有描述但非布尔判定 | 每操作有布尔判定("X成立=通过,不成立=失败") |
| D5 | 反幻觉(10%) | 无证据约束 | "请核实"但无路径 | 证据规则+不确定性标注+禁编造+来源引用格式 |
| D6 | 信息密度(10%) | ≥50%为寒暄/废话/短行 | 1-2段无实质 | 零寒暄/零废话/零≥3行连续<20字/每段提升有用功 |
| D7 | 边界清晰(10%) | 无边界 | 有做什么无"不做什么" | 做什么+≥3个不适用场景+≥3失败模式+每模式兜底 |
| D8 | 复用价值(10%) | 深度绑定特定上下文 | 可迁移,无模板/脚本 | 独立可用+含模板或脚本+无硬编码路径/token/凭据 |
| D9 | AI适配(10%) | 自然语言散文式 | 部分结构化,混模糊词 | 结构化输出模板+字段级定义+DeepSeek v4 Pro兼容(无模型特有指令) |
| D10 | 维护健康(5%) | >8KB或有过期引用 | 2-8KB,缺版本块 | 0.5-8KB+无过期引用+无硬编码token+版本块(版本号/日期/理由) |

**打分纪律**:
- D9 特别检查: 不含模型特有指令(如 `/slash` 命令、特定平台 `AGENTS.md` 等非通用引用) → 有则 ≤3 分
- D6 特别检查: 统计连续 <20 字的行数 → ≥3 行连续短行扣 ≥2 分
- D10 特别检查: >8KB 自动扣至 ≤5 分

## 扫描流程

1. **Step 0 预检**: 跑检查清单，任一❌中止
2. **Step 1 环境验证**: 统计 N = SKILL.md 数量。N=0 中止
3. **Step 2 逐文件评分**: 读 frontmatter + body → 10 维独立打分 → 每维 ≤30 字评语。读失败=INC
4. **Step 3 自动分类**: 按 name 前缀(首个 `-` 前字符)分组；无前缀归"未分类"
5. **Step 4 排序+分级**: 总分=Σ(维度×权重)，降序。S(≥85) A(75-84) B(65-74) C(55-64) D(<55) INC
6. **Step 5 历史对比**(如存在上次报告): 读取 `<dir>/_reports/` 中最近一次报告 → 逐 skill 对比总分 → 标记"↑提升/↓退化/→持平"，退化>5分标红
7. **Step 6 收费产品门禁判定**: 对每个 skill 判定是否达到"可对外收费发布"标准(见下方门禁)
8. **Step 7 输出报告**: 严格按下述模板，写入 `<dir>/_reports/skill-review-YYYYMMDD-HHmm.md`

## 收费产品级质量门禁（6 条全过=可收费发布）

| # | 门禁条件 | 判定 |
|---|---------|------|
| G1 | 总分 ≥80 | 达标/未达标 |
| G2 | D1(触发精准) ≥8 | 达标/未达标 |
| G3 | D3(可执行性) ≥8 | 达标/未达标 |
| G4 | D6(信息密度) ≥8 | 达标/未达标 |
| G5 | D7(边界清晰) ≥8 | 达标/未达标 |
| G6 | body≤8000 bytes 且无硬编码凭据 | 达标/未达标 |

**判定**: 6/6 = 可收费发布 | 4-5/6 = 需 1-2 轮优化 | ≤3/6 = 不可发布，需重设计

## 自动修复建议生成（R6新增）

对评分 <80 的 skill，自动生成可执行的修复 patch（不自动应用，仅输出建议）：

### 自动修复模板(7条)
| 症状 | 触发 | 修复 |
|------|------|------|
| 触发词不足 | D1<7 | body提取3-5动词+名词→追加到触发词列表 |
| 缺不适用场景 | D7<6 | 分析相邻skill→草拟2-3条不适用场景 |
| 缺失败兜底 | D7<6 | 每步追加"如果失败→"兜底句 |
| 缺版本块 | D10<5 | body末尾追加版本模板(版本号+日期占位) |
| 硬编码凭据 | D10检查 | 定位行→替换为`$env:VAR`占位符 |
| 缺布尔验证 | D4<6 | 每操作追加"验证:[布尔条件]" |
| 连续短行 | D6<8 | 合并<20字连续短行为单行(—分隔) |

### 修复建议输出格式
```markdown
## 自动修复建议
| Skill | 问题 | 优先级 | 建议操作 | 修改位置 |
|-------|------|--------|---------|---------|
| name | D7=4 缺兜底 | P0 | 在第X步后追加"如果失败→" | L45-L50 |
```

## 定期巡检建议
- **高频 skill**(每周使用≥5次): 每 2 周评测一次
- **中频 skill**(每周 1-4 次): 每月评测一次
- **低频 skill**(每月 <4 次): 每季度评测一次，连续 2 季度不使用 → 建议归档

## 失败兜底
| 失败 | 动作 |
|------|------|
| 目录不存在/不可读 | 输出原因 → 中止 |
| 目录无 SKILL.md | 输出原因 → 中止 |
| _reports/ 不可写 | 降级输出到当前工作目录 |
| 单文件读失败 | 标记 INC → 继续 |
| frontmatter 解析失败 | D1=D2=0 → 继续 |
| 编码异常 | 尝试 UTF-8 → 失败则 INC |
| 上次报告不存在 | 跳过历史对比 → 标注"首次扫描" |

## 约束
- 只读不改: 禁止修改被评测文件
- 输出完整: 不因"篇幅"省略任何模块
- 评语可追溯: 扣分评语指向具体刻度
- 报告文件命名: `skill-review-YYYYMMDD-HHmm.md`
